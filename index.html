Mac OS Installation.

The Kidspeak software had a pre-requesite of the `Arduino` IDE, an `I2C OLED` Arduino Library, `Drivers` for the Arduino, `NodeJS` and the `Kidspeak software` itself.

A week before Leetspeak 2016, I discovered a bug in the Arduino IDE, so on the day of the event I was using an older version which I knew worked.

First you need to install Arduino 1.6.9.

Next you will need Winchiphead signed drivers for CH340-CH341, which currently work for Mac OS X 10.9 to macOS 10.12.

Download the OLED I2C Arduino library and install it.

If you do not already have NodeJS installed, download and install the latest stable version.

Finally, you need the Kidspeak software.

After you unzip it, open a Terminal, navigate to the Kidspeak application's folder and run `npm install && npm start` - after it is finished download all the npm packages, it will shortly say it is running on port 8080. Open a web browser to `http://localhost:8080` - please keep in mind that I have mostly tested it on Google Chrome, and although I have seen it run in Safari, I have also noticed some JavaScript related bugs on that browser. So usage on other browsers, I cannot promise will be a bug free experience right now.
